# Cairn 2.0 — P1b: Five Adapters (parallel) + Hardening/Packaging Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the five remaining tracker adapters (GitLab, Jira, Asana, Azure Boards, ClickUp) behind the proven contract suite, and land the hardening/packaging cluster from the P0+P1a final review.

**Architecture:** Phase A (sequential, shared files) hardens the core: contextual errors, input validation, pagination, a 60s read cache, an async dynamic-import registry, a working npx bin entrypoint, and committed dist for plugin installs. Phase B (five tasks, **parallel-safe** — disjoint files, registry untouched thanks to A5) adds one adapter module + unit tests + env-gated live contract test each. Phase C integrates: README status table, per-backend config examples.

**Tech Stack:** As P0+P1a — TypeScript 5, Node ≥ 20, zod, vitest, native fetch (DI-injected), `@modelcontextprotocol/sdk`.

## Global Constraints

- Base branch: `worktree-cairn-2` (stacked on PR #2; rebase onto `main` if PR #2 has merged).
- All paths below relative to `cairn/` (e.g. `server/src/…` = `cairn/server/src/…`). Do NOT modify cairn 1.x files (`commands/`, `skills/`, `hooks/`, `adapters/*.py`, `scripts/`) — the Python adapters are read-only REFERENCE material.
- Secrets: env-var values only; never on disk, never in logs or error messages. Each adapter reads its token from an env var named in config (with a documented default name).
- Issue states exactly `open | in_progress | closed`; `updatedAt` ISO-8601 UTC.
- Every adapter must pass `trackerContract` (server/test/contract.ts) unchanged. Live tests run ONLY when `CAIRN_LIVE_TESTS=1` AND that backend's env vars are set; skip cleanly otherwise; excluded from default `npm test`.
- Adapter module contract (established in Task A5): each `server/src/tracker/adapters/<type>.ts` exports `configSchema` (zod) and `make(config): Tracker`.
- All HTTP through `fetchJson` (server/src/tracker/http.ts) with a `context` label (Task A1). Adapters accept `(config, fetchImpl: FetchLike = fetch)` for fixture testing.
- List endpoints without Link-header pagination cap at 100 items and must `console.error` a truncation warning when exactly at the cap (no silent truncation).
- Conventional commits. `npx tsc --noEmit` and full `npx vitest run` green before every commit.

## Parallel execution model

- **Phase A (Tasks 1–7): sequential** in one worktree — they touch shared files (`http.ts`, `registry.ts`, `index.ts`, `package.json`).
- **Phase B (Tasks 8–12): parallel.** Each task creates only `server/src/tracker/adapters/<name>.ts`, `server/test/<name>.unit.test.ts`, `server/test/<name>.live.test.ts` — zero shared-file edits. Dispatch each in its own worktree (`isolation: worktree`), merge each behind unit+contract-fake+tsc green. Merge order irrelevant.
- **Phase C (Tasks 13): sequential** after all B merges.

```ts
const NEXT_RE = /<([^>]+)>;\s*rel="next"/;
const MAX_PAGES = 10;

export async function paginate(
  fetchImpl: FetchLike,
  firstUrl: string,
  init: RequestInit,
  opts: { retries?: number; backoffMs?: number; context?: string } = {},
): Promise<unknown[]> {
  const out: unknown[] = [];
  let url: string | undefined = firstUrl;
  for (let page = 0; url && page < MAX_PAGES; page++) {
    // Need headers, so inline a fetch here mirroring fetchJson's error mapping:
    // simplest correct implementation — call fetchImpl once per page through a
    // small internal variant of fetchJson that also returns the Link header.
    const { body, next } = await fetchPage(fetchImpl, url, init, opts);
    if (Array.isArray(body)) out.push(...body);
    url = next;
    if (url && page === MAX_PAGES - 1)
      console.error(`[cairn] pagination truncated at ${MAX_PAGES} pages for ${firstUrl}`);
  }
  return out;
}
```

```ts
export class ReadCache {
  private store = new Map<string, { at: number; value: unknown }>();
  constructor(private readonly ttlMs = 60_000) {}
  get<T>(key: string): T | undefined {
    const hit = this.store.get(key);
    if (!hit) return undefined;
    if (Date.now() - hit.at > this.ttlMs) {
      this.store.delete(key);
      return undefined;
    }
    return hit.value as T;
  }
  set(key: string, value: unknown): void {
    this.store.set(key, { at: Date.now(), value });
  }
  clear(): void {
    this.store.clear();
  }
}
```

```ts
import type { CairnConfig } from "../config.js";
import type { Tracker } from "./types.js";

const ADAPTER_PATHS: Record<CairnConfig["tracker"]["type"], string> = {
  github: "./adapters/github.js",
  gitlab: "./adapters/gitlab.js",
  jira: "./adapters/jira.js",
  asana: "./adapters/asana.js",
  "azure-boards": "./adapters/azure-boards.js",
  clickup: "./adapters/clickup.js",

export async function makeTracker(cfg: CairnConfig): Promise<Tracker> {
  const { type, config } = cfg.tracker;
  let mod: { configSchema: { safeParse(v: unknown): { success: boolean; data?: unknown; error?: { issues: Array<{ message: string; path: Array<string | number> }> } } }; make(c: never): Tracker };
  try {
    mod = await import(ADAPTER_PATHS[type]);
  } catch {
    throw new CairnError("CONFIG_INVALID", `tracker type '${type}' is not yet implemented`,
      "check cairn.json tracker.type, or update cairn");
  }
  const parsed = mod.configSchema.safeParse(config);
  if (!parsed.success) {

      `fix tracker.config in cairn.json (see templates/cairn.json.example)`);
  }
  return mod.make(parsed.data as never);
}
```

```ts
import { z } from "zod";
import { CairnError } from "../../errors.js";
import { fetchJson, paginate, type FetchLike } from "../http.js";
import type { Capability, Issue, IssueCreate, IssuePatch, IssueState, Phase, Tracker } from "../types.js";

const WIP = "in-progress";

export class GitLabTracker implements Tracker {
  readonly capabilities: Capability = { hasInProgress: true, hasPhases: true, hasDependencies: true, hasLabels: true };
  constructor(private cfg: z.infer<typeof configSchema>, private fetchImpl: FetchLike = fetch) {}

  private token(): string {
    const t = process.env[this.cfg.tokenEnv] ?? "";
    if (!t) throw new CairnError("AUTH_MISSING", `missing env ${this.cfg.tokenEnv}`, `export ${this.cfg.tokenEnv}=<gitlab token with api scope>`);
    return t;
  }
  private base(): string { return `${this.cfg.baseUrl.replace(/\/$/, "")}/api/v4/projects/${encodeURIComponent(this.cfg.project)}`; }
  private headers(): Record<string, string> { return { "PRIVATE-TOKEN": this.token(), "content-type": "application/json" }; }
  private api(method: string, path: string, body?: unknown, op = "api"): Promise<unknown> {
    return fetchJson(this.fetchImpl, `${this.base()}${path}`, { method, headers: this.headers(), body: body === undefined ? undefined : JSON.stringify(body) }, { context: `gitlab ${op}` });
  }
  private normalize(raw: { iid: number; title: string; description: string | null; state: string; labels: string[]; milestone: { id: number } | null; assignee: { username: string } | null; updated_at: string; web_url: string }): Issue {
    let state: IssueState = raw.state === "closed" ? "closed" : "open";
    if (state === "open" && raw.labels.includes(WIP)) state = "in_progress";
    return { id: String(raw.iid), title: raw.title, body: raw.description ?? "", state,
      labels: raw.labels.filter((l) => l !== WIP),
      phase: raw.milestone ? String(raw.milestone.id) : undefined,
      assignee: raw.assignee?.username, updatedAt: raw.updated_at, url: raw.web_url };
  }
  // createIssue: POST /issues { title, description, labels: csv, milestone_id? } — non-numeric phase → CONFIG_INVALID
  // getIssue(id): /^\d+$/ guard → GET /issues/:iid
  // updateIssue: PUT /issues/:iid — title/description/labels csv;
  //   state in_progress → labels = merge(current-normalized, WIP) (fetch-then-merge like github);
  //   state open → labels without WIP + state_event "reopen" only if currently closed is NOT tracked — always send labels sans WIP;
  //   state closed → state_event: "close"
  // closeIssue → updateIssue(id, { state: "closed" })
  // listIssues: paginate(`${base()}/issues?per_page=100&state=all`) → normalize → client-side phase/state filter
  // createPhase: POST /milestones { title } → { id: String(id), name: title, state: "open" }
  // listPhases: GET /milestones?state=all&per_page=100 (paginate) → map
}

export const configSchema = z.object({
  baseUrl: z.string().url().default("https://gitlab.com"),
  project: z.string().min(1),
  tokenEnv: z.string().default("GITLAB_TOKEN"),
  extraLabels: z.array(z.string()).default([]),
});
export function make(config: z.infer<typeof configSchema>, fetchImpl?: FetchLike): Tracker {
  return new GitLabTracker(config, fetchImpl);
}
```
